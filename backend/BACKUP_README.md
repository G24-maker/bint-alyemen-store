تم تثبيت نظام نسخ احتياطي تلقائي متقدم لقاعدة بيانات SQLite في متجر Bint Alyemen. يوفر هذا النظام:

- ✅ **نسخ احتياطي تلقائي يومي** - الساعة 2 صباحًا من كل يوم

## ⚙️ الإعدادات

### تغيير وقت النسخ الاحتياطي التلقائي

في ملف `server.js`، بحث عن السطر:

```javascript
cron.schedule('0 2 * * *', async () => {
```

**صيغة Cron:**
```
┌───────────── minute (0 - 59)
│ ┌───────────── hour (0 - 23)
│ │ ┌───────────── day of month (1 - 31)
│ │ │ ┌───────────── month (1 - 12)
│ │ │ │ ┌───────────── day of week (0 - 6) (Sunday to Saturday)
│ │ │ │ │
│ │ │ │ │
0 2 * * *
```

**أمثلة:**
- `0 2 * * *` - الساعة 2:00 AM كل يوم (الافتراضي)
- `0 0 * * *` - منتصف الليل (12:00 AM) كل يوم
- `0 */6 * * *` - كل 6 ساعات
- `0 2 * * 0` - الساعة 2:00 AM كل يوم أحد فقط

## 🔒 الأمان والسلامة

### ✅ ما يتم حفظه تلقائيًا:

1. **النسخ الاحتياطية التلقائية** - قاعدة البيانات الكاملة بما فيها:
   - بيانات المنتجات
   - بيانات الطلبات
   - بيانات العملاء
   - بيانات المسؤولين

2. **حماية البيانات:**
   - كل نسخة احتياطية مستقلة ومحمية
   - قبل استعادة، يتم حفظ النسخة الحالية كـ `store-before-restore-<timestamp>.db`
   - يمكن استعادة أي نسخة احتياطية قديمة

3. **التنظيف التلقائي:**
   - حذف النسخ الأقدم من 30 يوم
   - توفير مساحة تخزين
   - يمكن تعديل هذه المدة في `backup.js`

   - إضافة `node-cron` كـ dependency
   - إضافة أوامر `backup` و `backup:list`

## 🆘 استكشاف الأخطاء

### المشكلة: لا يعمل النسخ الاحتياطي

**الحل:**
```bash
# 1. تأكد من تثبيت node-cron
npm list node-cron

# 2. تأكد من وجود مجلد backups
ls -la backend/backups/

# 3. جرب النسخ اليدوي
npm run backup

# 4. افحص الأخطاء في السيرفر
npm run dev
```

### المشكلة: النسخ الاحتياطي كبير جداً

**الحل:**
- هذا طبيعي في البداية
- النسخ القديمة تُحذف تلقائيًا بعد 30 يوم
- يمكن حذف النسخ اليدوية من `backend/backups/`

### المشكلة: الاستعادة لم تعمل

**الحل:**
```bash
# 1. أوقف السيرفر (Ctrl+C)
# 2. نفذ الاستعادة
node backup.js restore backup-YYYY-MM-DD-HH-MM-SS.sqlite

# 3. ابدأ السيرفر مجددًا
npm start
```
